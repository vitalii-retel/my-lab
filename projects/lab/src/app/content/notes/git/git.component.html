Git notes Getting main branch into feature branch Merge git checkout feature/branch
git merge main It creates a new commit with changes from main branch. Safe but merging commit may look unnecessarily. More info

Rebase git checkout feature/branch
git rebase main It removes feature/branch commits, pulls changes from main branch and then recreates feature/branch commits. So feature/branch commits are always in the end of the branch. Since it recreates commits this approach should be used with local private branches only. If feature branch is pushed already - use merge instead. More info

Conclusion It looks safer to use merge all the time. Otherwise before using rebase it should always be checked if the branch is not pushed yet.
